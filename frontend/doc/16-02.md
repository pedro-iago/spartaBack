# Atualizações — 16-02

Registro das alterações, criações e motivos.

## Revisão de treino (/trainer/edit-workout — InstructorReview)

**Criações**
- Nenhum arquivo novo; tela totalmente refatorada para seguir o padrão do projeto (bg-page-dark, PageHeader, glass-card-3d, Input/Button do design system).
- Fluxo "Ajustar manualmente": estado editável dos sets (séries, reps, descanso), chamada a `trainingService.updateTraining()` com toast de sucesso/erro (sonner).
- Estado vazio quando não há `trainingId`/`training` no state: mensagem "Treino não encontrado" e botão "Ir para Solicitações".

**Alterações**
- Layout: container `max-w-5xl mx-auto`, `min-h-screen min-h-[100dvh] bg-page-dark`; PageHeader com leftSlot (botão voltar com ArrowLeft para /dashboard/professional/solicitacoes), rightSlot (link Admin se ADMIN + botão LogOut).
- Conteúdo: card do aluno (iniciais, nome, nível, foco, dias/semana, limitações); lista de sets em cards glass-card-3d com inputs controlados (séries, reps, desc. em segundos, dia em readonly).
- Rodapé fixo: botão "Ajustar manualmente" (ghost) chama `handleSaveManual` (PUT /trainings/{id}); botão "Aprovar treino" (primário) navega para Solicitações; ambos desabilitados apenas durante `saving`.
- FloatingNav removido desta tela para não sobrepor os botões e manter foco na revisão.
- Botão "Ajustar manualmente" deixou de ser desabilitado quando `sets.length === 0`; ao clicar sem sets, exibe toast informativo em vez de ficar travado; adicionado `type="button"` para evitar submit acidental.
- Uso exclusivo de lucide-react (removidos material-symbols e useSparta).

**Motivos**
- Alinhar a tela ao padrão visual e de navegação do projeto (Documentation.md).
- Permitir que o personal edite séries/reps/descanso e persista via backend (ajustar manualmente).
- Garantir que a tela seja acessível apenas a partir da lista de Solicitações, com dados do treino no state, sem depender de GET por id.
- Evitar 403 e travamento do botão; manter apenas um ponto de Sair (header), sem Sair no menu inferior nesta tela.

**Arquivos impactados**
- InstructorReview.tsx: refatoração completa (estado editável, updateSet, handleSaveManual, PageHeader, rodapé, remoção FloatingNav).
- ProfessionalSolicitacoes.tsx: navegação para /trainer/edit-workout passando `state: { trainingId: t.id, training: t }` nos botões Revisar e Aprovar.
