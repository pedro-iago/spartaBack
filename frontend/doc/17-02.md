## 17-02 — Autenticação completa (3 perfis) e tela de cadastro

- **Integração login backend + frontend**
  - Alinhado o login real com o backend Java, usando `authService.login` que consome `POST /auth/login` e recebe `{ name, role, token }`.
  - Ajustado o roteamento por perfil no front:
    - `STUDENT` → `/dashboard/student`
    - `PERSONAL` (personal trainer) → `/dashboard/professional`
    - `ADMIN` → `/dashboard/admin`
  - Atualizado o enum `UserRole` no front para usar `PERSONAL` (antes `PROFESSIONAL`), compatível com o enum `UserRole` do backend.

- **Níveis de acesso por role (front)**
  - Atualizado `PrivateRoute` em `App.tsx`:
    - Rotas de aluno: `allowedRoles={["STUDENT", "ADMIN"]}`.
    - Rotas de personal: `allowedRoles={["PERSONAL", "ADMIN"]}`.
    - Rotas de admin: `allowedRoles={["ADMIN"]}`.
  - Ajustado usos de role em dashboards/Admin:
    - `AdminDashboard` e `AdminUsers` exibem `"Personal"` quando o backend retorna `PERSONAL`.
    - `Dashboard` comum (`ui/modules/common/Dashboard.tsx`) decide qual painel renderizar com base em `UserRole.ADMIN | UserRole.PERSONAL | UserRole.STUDENT`.

- **Usuários padrão criados (backend)**
  - Criado `InitialUsersConfig` (seed automático ao subir a API), garantindo:
    - `professional@sparta.com` / senha `123` / role `PERSONAL`
    - `aluno@sparta.com` / senha `-` / role `STUDENT`
    - `admin@sparta.com` / senha `-` / role `ADMIN`
  - Criado teste utilitário `DefaultUsersPasswordGenerator` para recriar/validar os mesmos usuários via JPA:
    - Faz `findByEmail` e cria se não existir, com senha `123` codificada pelo `PasswordEncoder`.
    - Útil para ambiente de desenvolvimento sem precisar rodar SQL manual.

- **Tela “Criar Nova Conta” (frontend)**
  - Rota pública `/register` adicionada em `App.tsx`.
  - Refatorada a tela `Register.tsx` para seguir o mesmo visual do login:
    - Fundo escuro com logo `icon2.png` em marca d’água.
    - Card `glass-card-3d` com perspectiva 3D, sombra e cabeçalho com botão **Voltar** para `/login`.
    - Formulário com campos: nome completo, e-mail, senha e confirmação de senha.
    - Validação simples de senha (erro se `password !== confirmPassword`) e mensagem de erro visual em vez de `alert`.
  - Fluxo após submit:
    - `POST /auth/register` criando sempre um aluno (`role: "STUDENT"`).
    - Auto-login chamando `authService.login` e salvando `{ token, name, role, email }` em `localStorage` + `SpartaContext`.
    - Redireciona para `/goal` para continuar a anamnese.

- **Experiência do login inicial**
  - Mantida a tela de boas-vindas do login com dois CTAs:
    - **Acessar Conta** → step de login com e-mail/senha.
    - **Criar Nova Conta** → navega para a nova tela `/register`.
